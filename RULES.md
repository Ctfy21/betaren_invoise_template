# RULES.md — Инструкция для LLM-агента (Open Claw / Claude Code)

> Этот файл описывает, как LLM-агент должен формировать коммерческое предложение (КП) по запросу продажника. Агент использует базу знаний Obsidian (`ref_obs/`), данные (`ref/`), шаблон Typst (`main.typ`) и скрипт `generate.py`.

---

## 0. Триггер запуска

Когда пользователь произносит фразу **«Сформируй новое КП»** (или близкие варианты: «создай КП», «новое коммерческое предложение», «сделай КП»), агент **обязан** начать процедуру формирования коммерческого предложения по следующему алгоритму:

1. Спросить у продажника недостающие обязательные данные (см. раздел 1):
   - Имя клиента, регион — если не указаны в сообщении
   - Какие сорта и на какую площадь — если не указаны
   - Главную боль по каждому сорту — если не указана (или предложить автоподбор)
2. Прочитать релевантные статьи из wiki (см. раздел 2)
3. Подобрать сорта, если не указаны (см. раздел 3)
4. Озвучить рекомендацию и дождаться подтверждения
5. Сгенерировать PDF (см. раздел 4)

Если продажник сразу предоставил все данные в одном сообщении (например: «Сформируй новое КП для ООО Нива, Оренбургская область, ДФ 2020 на 1000 га, засуха»), агент может пропустить уточняющие вопросы и сразу перейти к шагу 2.

### Источник данных

**Агент берёт все справочные данные исключительно из папки `ref_obs/`** (Obsidian wiki). Это единственный источник истины для: сортов, урожайности, устойчивости к болезням, аргументов по болям, технологической карты, препаратов и цен. Агент **не должен** обращаться к `ref/varieties.json`, `ref/technology.json` или любым другим файлам для получения данных, пока пользователь явно не укажет иное. Файлы в `ref/` используются только скриптом `generate.py` при компиляции PDF.

## 1. Входные данные от продажника

Продажник предоставляет информацию в свободной форме (текст, голос, чат). Агент извлекает данные двух уровней:

### Шапка (один раз на клиента)

| Поле | Обязательное | Строка XLSX | Пример |
|------|:---:|:---:|--------|
| Имя клиента | да | B3 | ООО «Сельская жизнь» |
| Регион | да | B4 | Оренбургская область |
| Email клиента | нет | B5 | client@mail.ru |
| Телефон клиента | нет | B6 | +7 (495) 123-45-67 |
| Имя менеджера | нет | B7 | Иванов И.И. |
| Email менеджера | нет | B8 | manager@betaren.ru |
| Телефон менеджера | нет | B9 | +7 (495) 777-84-89 |

### Таблица сортов (строки 13+, каждая строка = отдельный сорт/поле)

| Столбец | Поле | Обязательное | Пример |
|:---:|------|:---:|--------|
| B | Сорт | да* | ДФ 2020 |
| C | Площадь (га) | да | 1500 |
| D | Главная боль | да* | Засуха |
| E | Комментарий | нет | Поле у реки |

У одного клиента может быть **несколько сортов на разных площадях**. Один XLSX = один клиент = один PDF с разделами по каждому сорту.

*Если продажник не указал сорт или боль — агент **подбирает сам** по алгоритму из раздела 3.

## 2. База знаний (ref_obs/)

Перед формированием КП агент **обязан прочитать** релевантные статьи из Obsidian wiki.

### Навигация по wiki

- **Точка входа:** `ref_obs/INDEX.md` — сводная таблица всех сортов
- **Карта связей:** `ref_obs/MOC.md` — связи по болям, регионам, технологии

### Какие файлы читать

| Ситуация | Файлы для чтения |
|----------|-----------------|
| Всегда | `ref_obs/MOC.md` |
| По выбранному сорту | `ref_obs/Сорта/{название_сорта}.md` |
| По боли «Засуха» | `ref_obs/Концепции/Засухоустойчивость.md` |
| По боли «Болезни» | `ref_obs/Концепции/Устойчивость к болезням.md` |
| По боли «Качество» | `ref_obs/Концепции/Классы качества пшеницы.md` |
| Для подбора сорта по региону | `ref_obs/Концепции/Регионы допуска.md` |
| Для технологической карты | `ref_obs/Технология/Технологическая карта.md` |
| Для сравнения вариантов | `ref_obs/Сравнения/Сравнение сортов по урожайности.md` |

### Доступные сорта

```
Володя | Изумруд Дубовицкого | ДФ 2020 | Система | Ермоловка | Сократ | Интеза
```

### Допустимые боли

```
Засуха | Болезни | Качество
```

## 3. Алгоритм подбора сорта (если не указан)

Если продажник не указал сорт, агент подбирает его по региону и боли:

**Шаг 1 — Фильтр по региону.** Прочитать `ref_obs/Концепции/Регионы допуска.md`, оставить только сорта, допущенные в указанный регион. Соответствие:

| Регион (ключевые слова) | Номер | Допущенные сорта |
|------------------------|:-----:|-----------------|
| Москва, Тула, Рязань, Центральный | 3 | Ермоловка, Сократ, Интеза |
| Воронеж, Курск, Тамбов, Черноземье | 5 | Володя, Изумруд Дубовицкого |
| Краснодар, Ростов, Ставрополь, Кавказ | 6 | Володя, Система, Интеза |
| Самара, Саратов, Пенза, Средневолж. | 7 | Володя, Изумруд Дубовицкого, ДФ 2020 |
| Оренбург, Челябинск, Урал | 9 | Изумруд Дубовицкого, ДФ 2020 |

**Шаг 2 — Ранжирование по боли.** Из отфильтрованных сортов выбрать лучший:

| Боль | Приоритет выбора |
|------|-----------------|
| Засуха | ДФ 2020 → Изумруд Дубовицкого → Система → Володя |
| Болезни | Система → Ермоловка → Володя → Интеза |
| Качество | Ермоловка → Володя → Интеза → Система |

**Шаг 3 — Если боль тоже не указана**, определить по региону:
- Регионы 7, 9 (восточные, засушливые) → боль = Засуха
- Регион 6 (Северо-Кавказский, разнообразный фон) → боль = Болезни
- Регионы 3, 5 (Центр, ЦЧР) → боль = Качество

**Шаг 4 — Озвучить выбор продажнику** перед генерацией:
> «Для Оренбургской области рекомендую сорт ДФ 2020 (доказанная засухоустойчивость: 56.7 ц/га на 2700 га). Фокус КП — засуха. Подтверждаете?»

## 4. Генерация КП

### Способ A — через generate.py (предпочтительный)

Агент создаёт заполненный XLSX и запускает скрипт:

```bash
# 1. Создать заполненный XLSX (шапка + таблица сортов)
python -c "
from openpyxl import Workbook
wb = Workbook(); ws = wb.active; ws.title='КП'
# Шапка
ws['B3'] = 'ООО \"Сельская жизнь\"'   # Имя клиента
ws['B4'] = 'Оренбургская область'       # Регион
ws['B5'] = 'client@mail.ru'              # Email клиента
ws['B6'] = '+7 (495) 123-45-67'          # Телефон клиента
ws['B7'] = 'Иванов И.И.'                 # Менеджер
# Таблица сортов (строка 13+)
ws['B13'] = 'ДФ 2020';     ws['C13'] = 1000; ws['D13'] = 'Засуха'
ws['B14'] = 'Володя';      ws['C14'] = 500;  ws['D14'] = 'Качество'
wb.save('input_filled.xlsx')
"

# 2. Сгенерировать PDF (один файл со всеми сортами)
python generate.py input_filled.xlsx -o КП_СельскаяЖизнь.pdf

# 3. Удалить временный файл
rm input_filled.xlsx
```

Результат: один PDF, содержащий:
- Обложка (клиент, регион, общая площадь)
- Для каждого сорта: карточка + технологическая карта с расчётом
- Сводная таблица со всеми сортами и общим итогом

### Способ B — прямая генерация data.json

Если generate.py недоступен или нужна кастомизация, агент может сформировать `data.json` напрямую и скомпилировать Typst.

**Шаг 1 — Прочитать данные:**
```bash
cat ref/varieties.json   # данные о сортах
cat ref/technology.json  # технологическая карта с нормами и ценами
```

**Шаг 2 — Рассчитать объёмы.** Формулы:

```
норма_высева = 220 кг/га
тонн_семян = площадь_га × 220 / 1000

Для каждого продукта в technology.json:
  если rate_basis == "seed":
    объём = rate × тонн_семян
  иначе:
    объём = rate × площадь_га

  стоимость = объём × price_per_unit

семена_стоимость = тонн_семян × 25000
итого = сумма_всех_стоимостей + семена_стоимость
```

**Шаг 3 — Собрать data.json** по точной схеме (см. раздел 5).

**Шаг 4 — Скомпилировать:**
```bash
typst compile main.typ output.pdf
```

## 5. Схема data.json

Файл `data.json` — единственный интерфейс между данными и Typst-шаблоном. Структура **фиксирована**.

Один клиент, массив `entries[]` — по одному элементу на каждый сорт/поле.

```jsonc
{
  // === Клиент (общее) ===
  "client_name": "string",
  "region": "string",
  "manager_name": "string",
  "email": "string",
  "phone": "string",

  // === Технология (общая для всех сортов) ===
  "technology": {
    "title": "Технология оптимальных решений",
    "crop": "Озимая пшеница",
    "seeding_rate": 220
  },

  // === Сорта (массив — один элемент на сорт/поле) ===
  "entries": [
    {
      "variety": {
        "name": "string",
        "biotype": "string",
        "quality_class": "string",       // "Сильная пшеница" | "Ценная пшеница" | "Пшеница-филлер"
        "yield_avg": 118.6,              // число или null
        "yield_max": 124.6,
        "yield_production": 85.4,
        "protein_percent": null,
        "gluten_percent": null,
        "key_advantages": ["string", ...],
        "registry_since": 2024,
        "region_names": ["string", ...]
      },
      "area_ha": 1500.0,
      "pain": "засуха",                  // "засуха" | "болезни" | "качество"
      "pain_label": "Засуха",
      "pain_argument": "string",
      "resistance": [
        {"disease": "string", "level": "string"}, ...
      ],
      "calculation": {
        "seasons": [
          {
            "name": "Перед посевом",
            "treatments": [
              {
                "num": 1,                // null если подкормка
                "phase": "до посева",
                "products": [
                  {
                    "name": "string",
                    "category": "string",
                    "rate": 200,
                    "unit": "кг/га",
                    "volume": 300000.0,  // = rate × area_ha (или × тонн_семян)
                    "price": 32,
                    "cost": 9600000.0    // = volume × price
                  }
                ]
              }
            ]
          }
          // ... "Осень", "Весенние обработки", "Летние обработки"
        ],
        "seeds": {
          "variety": "string",
          "volume_tonnes": 330.0,
          "price_per_tonne": 25000,
          "cost": 8250000.0
        },
        "grand_total": 70412850.0        // итого по этому сорту
      }
    }
    // ... ещё сорта
  ],

  // === Итоги (по всем сортам) ===
  "total_area": 2300.0,
  "total_cost": 107966370.0
}
```

## 6. Правила поведения агента

### Обязательные

1. **Всегда читай wiki перед генерацией.** Минимум: `MOC.md` + статья сорта + статья по боли.
2. **Не выдумывай данные.** Все цифры (урожайность, нормы, цены) берутся из `ref/varieties.json`, `ref/technology.json` или `ref_obs/`.
3. **Озвучивай рекомендацию до генерации.** Если подбираешь сорт/боль сам — сначала объясни выбор продажнику и дождись подтверждения.
4. **Считай точно.** Объёмы = норма × площадь (или × тонн семян для протравителей). Проверяй: `grand_total == сумма всех cost + seeds.cost`.
5. **Не меняй структуру data.json.** Шаблон `main.typ` ожидает фиксированную схему из раздела 5.

### Запрещённые

1. **Не редактируй main.typ** для конкретного КП. Все данные идут через data.json.
2. **Не редактируй ref/varieties.json или ref/technology.json** при генерации КП. Это справочные данные.
3. **Не используй сорта, отсутствующие в varieties.json.**
4. **Не меняй цены** без явного указания продажника.
5. **Не округляй промежуточные расчёты** — только итоговые суммы.

### Рекомендуемые

1. **Обогащай аргументацию из wiki.** `pain_argument` берётся из varieties.json, но при общении с продажником используй дополнительные данные из Obsidian (кейсы, сравнения, цифры устойчивости).
2. **Предлагай альтернативы.** Если в регионе доступно несколько сортов — кратко опиши отличия и дай выбрать.
3. **Называй стоимость на гектар.** `grand_total / area_ha` — ключевая метрика для клиента.
4. **Проверяй PDF после генерации.** Убедись, что файл создан и размер > 50 КБ.

## 7. Пример диалога

**Продажник:** Сделай КП для ООО Нива, Самарская область, 2000 гектаров, у них засуха и качество важно. Хотят два сорта.

**Агент (внутренняя логика):**
1. Читает `ref_obs/MOC.md` → Самара = регион 7 (Средневолжский)
2. Читает `ref_obs/Концепции/Регионы допуска.md` → регион 7: Володя, Изумруд Дубовицкого, ДФ 2020
3. Читает `ref_obs/Концепции/Засухоустойчивость.md` → ДФ 2020 лидер по засухе
4. Читает `ref_obs/Сорта/Володя.md` → протеин 18.2%, сильная пшеница

**Агент (ответ):**
> Для Самарской области рекомендую два сорта:
> 1. **ДФ 2020** (1200 га, фокус — засуха): 56.7 ц/га в засуху 2025, толерантен к клопу-черепашке
> 2. **Володя** (800 га, фокус — качество): протеин 18.2%, сильная пшеница
>
> Как разделить площади? Или предложить другую комбинацию?

**Продажник:** Да, 1200 и 800, генерируй

**Агент:** Создаёт XLSX с двумя строками, запускает `python generate.py ...`, отдаёт один PDF с обоими сортами + сводной таблицей.

## 8. Структура проекта (справочно)

```
betaren_invoise_template/
├── main.typ                 # Typst-шаблон (НЕ ТРОГАТЬ при генерации)
├── data.json                # Генерируемый файл данных для main.typ
├── generate.py              # Скрипт: XLSX → data.json → PDF
├── create_template.py       # Генератор пустого input_template.xlsx
├── input_template.xlsx      # Пустой шаблон для ручного заполнения
├── ref/                     # Справочные данные
│   ├── varieties.json       # 7 сортов: паспорт, урожайность, аргументы
│   └── technology.json      # Техкарта: 4 сезона, 8 обработок, ~25 препаратов
├── ref_obs/                 # Obsidian wiki (база знаний)
│   ├── INDEX.md             # Главный индекс
│   ├── MOC.md               # Карта связей
│   ├── raw/                 # Исходные документы
│   ├── Сорта/               # 7 статей по сортам
│   ├── Концепции/           # 6 концептуальных статей
│   ├── Технология/          # 7 статей по препаратам
│   └── Сравнения/           # 2 сравнительных анализа
├── logo.png                 # Изображения для шаблона
├── decor-header.png
├── decor-body.png
└── team-photo.png
```
